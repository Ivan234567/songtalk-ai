# Описание конечного продукта: SongTalk AI (Speakeasy)

Платформа для изучения английского языка через **интерактивные песни (караоке)** и **разговорную практику с ИИ-собеседником**: голосовой и текстовый диалог, ролевые сценарии, дебаты, личный словарь с повторениями и аналитика прогресса. Реализована монетизация по факту использования (баланс, списание за запросы).

---

## 1. Назначение платформы

- **Цель:** помочь пользователям преодолеть языковой барьер и развить устную речь на английском в комфортной среде (без страха ошибок, без привязки к репетитору).
- **Форматы:** караоке по YouTube-песням с субтитрами и словарём; голосовой/текстовый диалог с ИИ; ролевые сценарии (ситуации); дебаты на заданную тему; личный словарь с SRS-повторениями; аналитика прогресса по баллам и критериям.
- **Доступ:** веб-приложение (Next.js), авторизация через Supabase Auth; оплата — пополнение баланса, списание за использование (чат, TTS, STT, словарь и т.д.).

Подробнее о проблеме и решении — в [docs/zayavka.md](zayavka.md).

## 2. Разделы приложения (навигация)

После входа пользователь попадает в **дашборд** с боковым меню. Доступные разделы:

| Раздел | Описание |
|--------|----------|
| **Главная** | Сводка: минуты разговоров, streak (дни подряд), счётчики песен/словаря, быстрые действия (Караоке, Словарь, Собеседник), недавняя активность. |
| **Караоке** | Загрузка YouTube-видео, синхронизированные субтитры, подсветка текущей строки, добавление слов/идиом/фразовых глаголов в словарь. |
| **Словарь** | Слова, идиомы и фразовые глаголы: просмотр, фильтры, категории, повторение (SRS), экспорт/импорт (CSV, JSON, Anki). |
| **Собеседник** | Голосовой и текстовый ИИ-партнёр: свободный диалог, ролевые сценарии, дебаты; подсказки ответа, переводчик, чат с ИИ. |
| **Прогресс** | Аналитика: средний балл, тренды, критерии оценки, последние сессии, рекомендации сценариев для повторения, страница фокуса по сценарию/теме. |
| **Пополнение баланса** | Текущий баланс, кнопки пополнения (300 / 500 / 1 000 ₽), статистика использования за период, история расходов, экспорт CSV. |

## 3. Караоке

**Назначение:** изучение языка через YouTube-видео с синхронизированными субтитрами и интеграцией со словарём.

### 3.1 Основные возможности

- **Источник:** YouTube (ссылки `youtube.com/watch?v=...`, `youtu.be/...`, `youtube.com/embed/...`).
- **Субтитры:** загрузка через бэкенд (yt-dlp): ручные и автосгенерированные, формат SRT; парсинг в сегменты (время начала/конца, текст).
- **Плеер:** YouTube IFrame API; скорость 0.5x–2x, громкость, полноэкранный режим.
- **Режимы просмотра:** полный (видео + текст), только текст (режим караоке).
- **Синхронизация:** подсветка текущего сегмента по времени воспроизведения, автоскролл к активной строке; переход к сегменту по клику.

### 3.2 Управление

- Play/Pause — пробел; назад/вперёд 5 сек — стрелки ←/→; скорость — ↑/↓; звук — M.
- Сохранённые видео: список загруженных, загрузка в плеер, удаление с подтверждением.

### 3.3 Интеграция со словарём

- **Слова:** клик по слову → запрос перевода/определения → добавление в словарь с контекстом (текст сегмента); TTS-озвучка.
- **Идиомы:** кнопка «Анализ идиом» → разбор текста песни (до ~20 идиом): фраза, дословный перевод, значение, примеры; добавление в словарь одним кликом.
- **Фразовые глаголы:** аналогичный анализ и добавление с привязкой к видео.

### 3.4 API (бэкенд)

- `POST /api/karaoke/transcribe` — загрузка субтитров YouTube, парсинг SRT, сохранение в БД.
- `GET /api/videos`, `GET/DELETE /api/videos/:id` — список и управление видео пользователя.
- Добавление в словарь: см. раздел «Словарь», API словаря.

## 4. Собеседник (AI-агент)

**Назначение:** голосовой и текстовый ИИ-партнёр для практики английского в трёх режимах — свободный диалог, ролевые сценарии, дебаты. Есть подсказки ответа, панель переводчика и отдельный чат с ИИ.

### 4.1 Режимы работы

#### Свободный режим (Freestyle / Chat)

- Неструктурированный разговор: пользователь говорит (запись → STT) или печатает; ответ ИИ через LLM (streaming), озвучка через TTS.
- **Настройки:** подсказки ответа (естественно, проще, нативно, вежливо, без мата); сленг (off / light / heavy); 18+ (разрешён ли мат в контексте, может ли ИИ использовать мат, интенсивность); формальность и прямота (0–100); подсказка роли (например «собеседник в кафе»).
- Сессии сохраняются, отображаются в истории и учитываются в аналитике.

#### Ролевые сценарии (Roleplay)

- Диалог в заданной ситуации (такси, отель, ресторан и т.д.) с целью и шагами.
- **Источники сценариев:** каталог (встроенные), «Мои сценарии» (пользовательские), создание с помощью ИИ.
- Брифинг → первая реплика персонажа (опционально с озвучкой) → поочерёдный обмен репликами; ИИ помечает выполненные шаги; подсказка ответа с учётом шагов; по завершении — «Отметить цель выполненной» → фидбек и полезные фразы; опционально оценка речи.
- У каждого сценария свои настройки сленга и мата.

#### Дебаты (Debate)

- Пользователь и ИИ занимают противоположные позиции по теме.
- Выбор темы (каталог, своя тема, «Мои темы»), позиция (За/Против), сложность (easy/medium/hard), микроцели, кто начинает.
- Брифинг → поочерёдные реплики; подсказка ответа; по завершении — фидбек в формате SBI (Situation, Behavior, Impact), сильные стороны, зоны роста, полезные фразы; опционально оценка речи.
- Своя тема валидируется и нормализуется через ИИ (`/api/agent/debate-topic-prepare`).

### 4.2 Подсказка ответа (Reply Hint)

- `POST /api/agent/reply-hint`: ИИ предлагает вариант ответа пользователя по последней реплике. Работает в режимах roleplay (с учётом шагов и цели), debate (позиция, аргументы), chat (формальность, прямота). Учитываются настройки сленга и мата.

### 4.3 Переводчик и чат с ИИ

- **Панель переводчика:** EN ↔ RU через LLM; перетаскиваемое окно; история переводов; выделение слова/фразы → добавление в словарь (AI определяет тип: word/idiom/phrasal_verb и сохраняет в соответствующий раздел).
- **Чат с ИИ:** отдельное перетаскиваемое окно для любых вопросов и уточнений; несколько сессий, создание/загрузка/удаление; сохранение в БД.

### 4.4 Создание контента с помощью ИИ

- **Ролевые сценарии:** `POST /api/user-scenarios/generate` — по свободному тексту (тема, место, роль, цель, уровень A1–C1, сленг/мат) ИИ возвращает JSON сценария; пользователь редактирует и сохраняет в «Мои сценарии».
- **Темы дебатов:** `POST /api/agent/debate-topic-prepare` — валидация, нормализация, рекомендованная сложность; сохранение в «Мои темы».

### 4.5 Оценка речи

- `POST /api/agent/assess-speaking` — баллы по критериям (fluency, grammar, vocabulary, pronunciation, coherence и др.) и рекомендации. Привязывается к завершениям ролевых сценариев и дебатов.

### 4.6 API (бэкенд)

- `POST /api/agent/stt` — распознавание речи (Whisper).
- `POST /api/agent/tts` — озвучка текста (TTS).
- `POST /api/agent/chat` — основной чат, streaming NDJSON.
- `POST /api/agent/reply-hint` — подсказка ответа.
- `POST /api/agent/roleplay-feedback`, `POST /api/agent/debate-feedback` — фидбек по сценарию/дебату.
- `POST /api/agent/debate-topic-prepare` — валидация темы дебата.
- `POST /api/agent/assess-speaking` — оценка речи.
- `GET/POST/PATCH/DELETE /api/user-scenarios` — CRUD личных сценариев.

## 5. Словарь

**Назначение:** персональная коллекция слов, идиом и фразовых глаголов с переводами, транскрипциями, примерами, контекстами из песен и повторением по алгоритму SRS (интервальное повторение).

### 5.1 Три раздела

- **Слова** (`user_vocabulary`): слово, переводы, уровень сложности (A1–C2), часть речи, уровень освоения (1–5), счётчики просмотров/практик, контексты из видео, дата следующего повторения. Источники: караоке (клик по слову), переводчик (выделение → добавление), импорт (CSV, JSON, Anki).
- **Идиомы** (`user_idioms`): фраза, дословный перевод, значение, примеры, привязка к видео. Источники: караоке («Анализ идиом»), переводчик.
- **Фразовые глаголы** (`user_phrasal_verbs`): аналогично идиомам. Источники: караоке («Анализ фразовых глаголов»), переводчик.

### 5.2 Кэш определений

- Таблица `word_definitions_cache` (общая): при добавлении слова сначала поиск в кэше; при отсутствии — запрос к LLM, кэширование, списание баланса.

### 5.3 Интерфейс

- Режимы просмотра: слова / идиомы / фразовые глаголы.
- Фильтры: поиск (с задержкой), уровень сложности (A1–C2), категория.
- Сортировка слов (по уровню сложности).
- Панель деталей: переводы, транскрипция, примеры, контексты из песен, TTS, категории, удаление.
- Статистика: всего слов, слов на повторение.

### 5.4 Повторение (SRS)

- `GET /api/vocabulary/review-list` — слова на повторение (next_review_at ≤ сейчас, лимит по умолчанию 20).
- `POST /api/vocabulary/review` — отправка результата (слово, score 0–1); пересчёт next_review_at, обновление mastery_level и vocabulary_progress.

### 5.5 Категории

- Категории (название, описание, цвет, иконка) с привязкой к словам, идиомам и фразовым глаголам. API: GET/POST/PUT/DELETE категорий, назначение категорий элементам словаря.

### 5.6 Экспорт и импорт

- **Экспорт:** `GET /api/vocabulary/export` — форматы CSV, JSON, Anki; параметры: view_mode, search, difficulty, category.
- **Импорт:** `POST /api/vocabulary/import` — format (csv, json, anki), data; автоопределение Anki по полям Front/Back.

### 5.7 API (основные)

- Определение и добавление: `/api/vocabulary/define`, `/api/vocabulary/add`; идиомы: analyze, add, list, delete, bulk-delete; фразовые глаголы: аналогично.
- Список и удаление слов: list, word/:id DELETE, words/bulk-delete.
- Повторение: review-list, review.
- Категории и привязки: см. выше.

## 6. Прогресс (аналитика)

**Назначение:** панель прогресса по практике устной речи — баллы, тренды, критерии, последние сессии, рекомендации для повторения. Данные из завершений ролевых сценариев и дебатов и из оценок речи (`speaking_assessments`).

### 6.1 Фильтры

- **Период:** 7 / 30 / 90 дней, всё время (горячие клавиши 1–4).
- **Режим:** ролевые сценарии / дебаты (R / D).
- **Тип контента:** системный (каталог) / личный (S / L).

### 6.2 Структура экрана

- **ProgressHero:** средний балл (кольцо из 10), число сессий за период, слабый критерий, дельта к предыдущему периоду, streak (дни подряд), график тренда (средний балл по дням/неделям); кнопки «Начать практику», «Повторить слова».
- **KPI-карточки:** средний балл, число сессий, лучший балл (с дельтами и спарклайнами).
- **Критерии оценки:** бары или радар по критериям (fluency, vocabulary_grammar, pronunciation, completeness, dialogue_skills); выбор фокус-критерия для рекомендаций.
- **Последние сессии:** список завершений (ролевой/дебаты), название, балл, дата; сортировка по дате/баллу/типу; клик → страница фокуса.
- **Рекомендации сценариев:** на основе слабого (или выбранного) критерия; приоритет high/medium/low по баллу; «Практиковать сейчас», «Открыть разбор».

### 6.3 Страница фокуса

- URL: `/dashboard/progress/focus/[objectKey]` (objectKey: `rp:{scenario_id}` или `db:{topic_key}`).
- Детальный разбор по сценарию или теме дебата: все попытки за период, баллы по критериям, шаги и цели, сильные стороны, улучшения, комментарий тренера, расшифровка реплик.

### 6.4 Метрики на главном дашборде

- Минуты разговоров (оценка по сообщениям), сегодня/за неделю, streak (дни подряд с активностью: агент, ролевые, дебаты, словарь).

## 7. Пополнение баланса и монетизация

**Назначение:** оплата по факту использования. Баланс в рублях; списание при платных запросах (чат, TTS, STT, определение слов и т.д.); пополнение — ручное (админ) или через платёжный шлюз (ЮKassa, по плану).

### 7.1 Что видит пользователь

- Текущий баланс.
- Кнопки пополнения: 300 ₽ (~2 часа с агентом / до 800 озвучек), 500 ₽ (~3,5 часа / до 1 300 озвучек), 1 000 ₽ (~7 часов / до 2 500 озвучек). Минимум пополнения — 300 ₽.
- Переключатель периода: сегодня / 7 / 30 / 90 дней.
- Сводка за период: сумма расходов, количество запросов, среднее в день, среднее за запрос.
- Топ по тратам и по запросам (по моделям/сервисам).
- История расходов: модель/сервис, дата, сумма (руб.).
- Экспорт истории в CSV.

### 7.2 Тарифы (для справки)

- Наценка 1,5× к закупке; в цену заложен налог УСН 6%. В интерфейсе пользователю показываются не технические единицы (токены/символы), а «что получишь» (минуты, озвучки). Детали — в [docs/monetization-plan.md](monetization-plan.md).

### 7.3 Платные действия

- Собеседник: chat, TTS, STT; переводчик; анализ/добавление в словарь через ИИ; генерация сценариев, подготовка темы дебата; оценка речи.
- Караоке: транскрипция (если платная), определение слов.
- Словарь: определение слова (при промахе кэша), анализ идиом/фразовых глаголов.

При недостаточном балансе возвращается 402 и призыв пополнить баланс.

## 8. Авторизация и технический стек

- **Авторизация:** Supabase Auth (JWT). Бэкенд проверяет JWT; баланс читается из БД при платных запросах и на странице баланса.
- **Frontend:** Next.js 14 (App Router), деплой на Vercel.
- **Backend:** Node.js + Express, деплой на Railway.
- **БД и real-time:** Supabase (PostgreSQL + Realtime при необходимости).
- **Внешние сервисы:** AITUNNEL (OpenAI-compatible) для LLM, TTS, STT; yt-dlp для субтитров YouTube.
